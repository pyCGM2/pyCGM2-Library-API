Detailed Description

Compute kinematic angles using the joint coordinate system (JCS) proposed by Grood & Suntay (1983) as well as global angles.

This filter gives you the possibility to choose the axes sequence for each joint. For this, you have to use the method SetRotationSequence(). For example:

// model, arm and forearm already defined
btk::Joint::Pointer jnt = btk::Joint::New(forearm, hand, "Wrist");
model->AppendJoint(jnt);
btk::ModelJCSKinematicFilter::Pointer mjkf = btk::ModelJCSKinematicFilter::New();
mjkf->SetInput(model);
mjkf->SetRotationSequence("Wrist", btk::AxisZ, btk::AxisY, btk::AxisX); // Wrist kinematic using the sequence Z-Y'-X''
btk::Acquisition::Pointer output = mjkf->GetOutput();
output->Update();
// Wrist angles are in the point labeled WristAngle.

The default sequence is Z-X'-Y'', then you don't have to set the sequence for each joint (if you want to calculate the kinematic with the sequence is Z-X'-Y'').

Moreover, even, if it is advised to give a unique label for each joint, you can append the same joint many times and look differences in the kinematics when choosing different sequences. For example:

// model, arm and forearm already defined
btk::Joint::Pointer jnt = btk::Joint::New(arm, forearm, "Elbow");
model->AppendJoint(jnt);
model->AppendJoint(jnt); // Added a second time
btk::ModelJCSKinematicFilter::Pointer mjkf = btk::ModelJCSKinematicFilter::New();
mjkf->SetInput(model);
mjkf->SetRotationSequence(0, btk::AxisZ, btk::AxisX, btk::AxisY); // Elbow kinematic using the sequence Z-X'-Y''
mjkf->SetRotationSequence(1, btk::AxisZ, btk::AxisY, btk::AxisX); // Second elbow kinematic using the sequence Z-Y'-X''
btk::Acquisition::Pointer output = mjkf->GetOutput();
output->Update();
// Angles can be compared by comparing values in the points "ElbowAngles" and "ElbowAngles2".

As presented previously in the code, you can set the sequence by using the joint's label or its index in the model. If more than one joint use the same label, then all these joints will use the given sequence. Sequences given by index have priority on sequences given by label. Then if you give for the same joint a sequence by using the joint's label and the join't index used in the model, the final selected sequence will be the second.

If the model has the option 'btk::CompatibleVicon', then the angles' label will be finished by "Angles" instead of "Angle".

Detailed Description

Object representing the modeling of a body part.

A segment is represented by:

  • a label and a description ;
  • a type (a tag to indicate the body part) ;
  • a side ;
  • a technical frame (TF) expressed in the global frame (also named technical coordinate system: TCS) ;
  • an segmental frame (SF) expressed in the technical frame (also named segmental coordinate system: SCS) ;
  • an inertial frame (IF) expressed in the segmental frame (also named inertial coordinate system: ICS) ;
  • a tensor of inertia expressed in the segmental frame (SF) or the inertial frame (IF) ;
  • a center of mass expressed in SF or IF ;
  • a mass in kilograms ;
  • a length in millimeters (distance between the end points) ;
  • a map of label and Position expressed in SF ;
  • a list of external wrenches including the proximal and distal joint wrenches.

The motion of the segment is computed in the global frame an stored as the motion of TF in the global frame. You can access to this data by using the method GetTechnicalFrameMotion(). To extract the motion of the segmental frame or the inertial frame, you can use the method GetSegmentalFrameMotion() and GetInertialFrameMotion() respectively. In both case these methods compute dynamically the motion of each reference frame from the technical frame and the relative rigid transformations set.

An enumeration of segment's type is provided (see Segment::Type). However, if the desired type is not proposed you can easily extend the list by using the value Segment::User. For example:

int HAT = Segment::User + 1; // Head and Trunk
enum {Patella = Segment::User + 2, Metatarsus, Tarsus}

Constructor. Use the enumeration Segment::Type to define the type t of the segment.

The proximal and distal joint wrenches are set to null. You must use the methods Segment::SetProximalJointWrench and Segment::SetDistalJointWrench to define the wrench. You can use a Wrench object for the proximal joint and a WrenchReaction object for the distal joint. This latter being coupled with the proximal joint of the previous segment. By using this solution, it is not necessary to store redundant information. It has also the benefit to reduce memory consumption.

Wrench::Pointer btk::Segment::GetDistalJointWrench ( )
inline

Convenient method to call GetExternalWrench(btk::SegmentJointDistalWrench) and return the distal joint wrench. By default the distal joint wrench doesn't exist. Calling this method in this case has for effect to create an invalid wrench (residual of the position set to -1). Be sure to adapt the content of the residual, otherwise the inverse dynamic won't take into account the value. Another way to set the distal joint wrench is to call SetDistalJointWrench().

void btk::Segment::SetRelativeInertialFrameFromSegmentalFrame ( const Position &  origin,
AnatomicalDirection  segmentalAxisX,
AnatomicalDirection  segmentalAxisY,
AnatomicalDirection  segmentalAxisZ 
)

Set the relative inertial frame from the given anatomical directions to go through SF to IF.

The inertial frame has its axes going in the following directions (when looking from the rear):

  • X: to the right direction (sagittal axis)
  • Y: to the forward direction (transversal axis)
  • Z: to the up direction (longitudinal axis) If the segmental frame is oriented in another way, then it is important to provide the relative orientation between SF and IF. In this method, the given directions correspond to the anatomical direction used by the segmental frame (SF). Then, they are converted to find the transformation between SF and IF.

The given origin is the position of the origin of the inertial frame relative to the segmental frame.

Warning: The directions for the axes X,Y,Z must be different and representing the tree major directions (AP|PA, ML|LM, PD|DP). For example:

// SF defined as following: axis X: right to left, axis Y back to front, axis Z: up to bottom
// Then, you need to provide the direction used by SF for each of these axes.
seg->SetRelativeInertialFrameFromSegmentalFrame(btk::Segment::LM, btk::Segment::PA, btk::Segment::PD);
